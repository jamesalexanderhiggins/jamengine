Build & Deploy — Operator's Instruction Sheet
What this document is: A step-by-step, phase-by-phase guide to building and deploying a Jam Engine instance — for yourself, or for a new client. Written for a non-technical operator who works with Claude to do the building.

How to use it: Work through each phase in order. Each step has a checkbox — click it when done. Steps are colour-coded by who does the work: you, Claude, or both together. When it says Claude, open a new chat, paste the step, and let it run.

Save your API keys somewhere safe — you'll give them to Claude in Phase 3

5
Import Your Songs
10–30 min depending on catalog size
You
Option A: Import from Songbook Pro (.sbp file) +
  1. In Songbook Pro on your device, go to File → Export → Export as .sbp
  2. Email the .sbp file to yourself or AirDrop it to your computer
  3. In your Jam Engine admin, go to Songbook → Import Songs
  4. Select "Songbook Pro (.sbp)" and upload the file
  5. Your entire catalog imports — songs, keys, and setlists
  6. Review imported songs and set default keys for any that need adjusting
Option B: Import from Ultimate Guitar (paste method) +
  1. On Ultimate Guitar, find the song and open the chord tab
  2. Select all the text (Ctrl+A or Cmd+A) and copy
  3. In Jam Engine admin, go to Songbook → Add Song → Paste from UG
  4. Paste into the text box — the app parses and formats it automatically
  5. Set the artist name, song name, original/cover, and default key
  6. Save. Repeat for each song.
Option C: Enter songs manually +

In the Song Editor, type your chord chart in the same format as Ultimate Guitar — chords in square brackets inline with lyrics, section headers like Verse / Chorus / Bridge. The editor has a live preview so you can see it formatted as you type.

Run a Test Show
20 minutes · Do this before your first real gig
You + a mate
⚠ Do Not Skip This Running a test show before your first real performance saves embarrassment. It takes 20 minutes and you'll feel completely confident by the end of it.
○
STEP 8.1
Create a test show
In your admin, go to Shows → Create Show. Pick a test venue name, today's date, choose your setlist, set your tip pricing options, and hit "Start Now". Confirm the "Are you ready to go?" popup. Your show is now live.

STEP 8.2

STEP 8.3
Send a test tip and request
Use Stripe's test card (4242 4242 4242 4242) on the fan view to send a $5 test request. Watch it appear in your performer view in real time. Try marking it as played and see the song colour change on both screens.
Use test mode
○
STEP 8.4
Check the Coming Up Next display mode
Open [yourname].jamengine.com/display on a tablet or laptop. This is the fullscreen queue display for venue TVs. Confirm the queue updates when you change it on the performer view.

End the test show and check show history
In your admin, click "End Show" → confirm. Then go to Analytics → Show History. You should see the test show with your test tip recorded. This is how all your show history will be saved going forward.

When Things Go Wrong — Quick Fixes

"The site isn't loading / showing an error" +

Open Claude, describe what you see: "My Jam Engine site at [url] is showing [error message]. Here are my Vercel environment variables [list them]. What's wrong?" Claude will diagnose and give you the fix.

"Stripe payments aren't going through" +

First check: is test mode on or off? (Settings → Payments → toggle). If test mode is off and you're using a real card, check your Stripe dashboard for the failed payment and read the error code. Paste the error code into Claude and it will tell you what to fix.

"The real-time queue isn't updating for my band" +

This is almost always a Supabase Realtime configuration issue. Tell Claude: "My Supabase Realtime isn't syncing between devices. My Supabase URL is [url]." Claude will give you the exact settings to check in your Supabase dashboard.

"I want to change my branding / update a feature" +

For branding: Settings → Appearance. For feature changes: open Claude, describe the change ("I want to add X to the performer view / change Y to work like Z"), and Claude will generate the updated code and tell you where to paste it.
